### 一、Stage 2 的定位

**Stage 1 (监督学习预热)** 训练 FrontRES 网络预测 `[Δpos(3), Δrpy(3)]` 任务空间修正量，使网络初步学会从观测映射到修正动作。

**Stage 2 (RL 微调)** 在此基础上用 **PPO** 算法对 FrontRES 进行强化学习微调，通过与环境交互获得的奖励信号进一步提升修正精度和鲁棒性。

### 三、分模块详解

#### 3.1 环境配置 `FrontRESFinetuneTrackingEnvCfg` (`tracking_env_cfg.py` 第924-1155行)

继承自 `GeneralTrackingEnvCfg`，对 Stage 2 做了三个定制：

##### (a) 域随机化 `RLFinetuneEventCfg` (第936-1017行)

| 事件 | mode | 参数 | 说明 |
|------|------|------|------|
| `physics_material` | startup | static_friction: (0.6,1.5), dynamic: (0.5,1.2) | 地面材质（缩小范围，避免极端湿滑） |
| `add_joint_default_pos` | startup | ±0.02 rad | 编码器零点误差 |
| `base_com` | startup | x:±0.05, y:±0.05, z:±0.05 m | 质心偏移 |
| `randomize_actuator_properties` | startup | stiffness/damping: ±15% | PD 增益标定误差 |
| `add_payload` | startup | 0-2 kg | 负载（G1 ≈ 35kg, +5.7%） |
| `push_robot` | interval | 4.0-8.0s, 降低推力强度 + 频率 | 真实扰动 |

**设计原则**：
- **全部使用 `mode="startup"`**：物理属性在仿真启动时固定，Critic 能为每个 env 学出稳定的 V(s)
- **删除重力随机化**：现实中重力恒定，纯噪声
- **收窄范围**：最坏组合概率约 0.03%（8192 env 中仅 2~3 个）

##### (b) 奖励函数 `RLFinetuneRewardsCfg` (第1019-1129行)

与基础奖励 `RewardsExpertCfg` 的主要区别：

| 奖励项 | weight | 说明 |
|--------|--------|------|
| `contact_feet` | 1.0 | **新增** — 脚-地面接触匹配（二值：1/0） |
| `motion_feet_pos` | 1.0, σ=0.05 | **新增** — 脚部位置精度（σ=5cm，严格） |
| `motion_body_pos` | 0.5, σ=0.3 | 全身肢体位置对齐 |
| `motion_body_ori` | 0.5, σ=0.4 | 全身肢体朝向对齐 |
| `motion_body_lin_vel` | 0.75, σ=1.0 | 全身线速度对齐 |
| `motion_body_ang_vel` | 0.75, σ=3.14 | 全身角速度对齐 |
| `motion_global_anchor_pos` | 0.5, σ=0.3 | 锚点（盆骨）位置 |
| `motion_global_anchor_ori` | 0.5, σ=0.4 | 锚点朝向 |
| `undesired_contacts` | -2.0 | 非预期接触惩罚（加重） |
| `joint_limit` | -20.0 | 关节限位惩罚（加重） |
| `joint_torque` | -2e-4 | 扭矩惩罚 |

**移除项**：`action_rate_l2` — 它惩罚的是 GMT 输出（robot_actions），而非 Δq 本身。Δq 平滑由 PPO 的 entropy + 自然探索隐式处理。

##### (c) Motion 级扰动 (第1137-1155行)

自适应 DR 的 base 值（Level 1，GMT 能轻松处理的强度）：

| 扰动类型 | prob | base value | 4× 全强度 |
|---------|------|-----------|-----------|
| float (上浮) | 0.3 | 0.05 ratio | 0.20 m |
| sink (下沉) | 0.3 | 0.04 ratio | 0.16 m |
| foot_slip (脚滑) | 0.2 | 0.008 ratio | 0.032 m |
| root_tilt (根倾斜) | 0.3 | 0.05 rad | 0.20 rad (~11.5°) |
| joint_noise (关节噪声) | 0.4 | 0.04 rad | 0.16 rad (仅下肢) |

##### (b) Checkpoint 加载

```

  student_checkpoint_path = stage1/model_10000.pt
  is_full_resume = False
  → 仅迁移 FrontRES 网络权重，优化器/学习率/噪声 std 重置

Stage 2 → Stage 2 断点续训:
  student_checkpoint_path = resume/model_XXXXX.pt
  is_full_resume = True
  → 恢复 Adam moments、学习率、噪声 std
```

##### (c) 自适应 DR (PI 控制器) (第219-243行)

核心思路：以 **per-step 存活率** 为反馈，连续调节 DR 倍率 `dr_scale ∈ [0, dr_max_scale]`。

| 参数 | 值 | 说明 |
|------|-----|------|
| `dr_target_episode_length` | 60 步 | 目标回合长度（≈半倍GMT基线125步） |
| `dr_adapt_speed` | 0.0005/iter | 每轮 dr_scale 最大变化量 |
| `dr_max_scale` | 4.0 | 上限，4×base = 原始最大扰动强度 |
| `dr_ema_alpha` | 0.95 | 存活率 EMA 平滑 |
| `dr_deadband` | 0.005 | 死区，避免频繁调整 |

**两阶段规划**：
- **Phase 1 (0-5000 iter)**：DR 从 0 线性爬坡
- **Phase 2 (5000-40000 iter)**：标准 PPO 全强度
